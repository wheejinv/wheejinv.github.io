Hexo 블로그 개설 시 저장소가 3개 필요한 이유

이 포스트에서는 Git 에 대해서는 설명하지 않고 있습니다.
Git 에 대해서는 생활코딩에 수준 높은 무료 강의가 있습니다.

저는 Git Repository 가 3개 필요했어요

결과적으로 Hexo와 관련된 총 3개의 Repository 를 생성했습니다.

  1. 빌드되어 나온 결과물이 올라가는 Repository.
  2. 실제 글을 쓰고, 각종 설정을 하는 Repository.
  3. 사용하는 테마의 Git Repository를 fork한 Repository.

그럼 각각의 브랜치가 필요한 이유에 대해 조금 더 자세하게 기술해보겠습니다.

배포를 하기 위한 Repository

해당 브랜치에는 빌드된 html 들이 올라가는 브랜치입니다.
만약 내 아이디가 wheejinv 라면 Repository 의 이름은 wheejinv.github.io 이런식으로 맞춰주세요.

Repository 명을 다르게 해서 셋팅하는 방법도 있지만 url 과 관련된 단점이 있는 것으로 알고 있습니다.

저의 깃 저장소 URL을 참고해주세요. https://github.com/wheejinv/wheejinv.github.io

실제 작업하는 Repository

마크다운 문법으로 글을 작성하고, 블로그에 대한 각종 설정을 하게 되는 주작업 Repository 입니다.
저는 저장소를 비공개로 하여 글의 원본은 공개하지 않도록 설정했습니다.

Hexo Init 을 하고.. _config.yml 파일 설정을 하고, 사용할 카테고리를 지정하는 등의 설정을 하게 됩니다.
참고: Hexo Setup

테마와 관련된 Repository

제일 필요 없는 Repository 라고 할수도 있지만 저는 필요하다고 생각합니다.
입맛대로 설정하다보면 수많은 설정들을 만지고 바꿔줘야 하는데 수정 내역이 없으면 나중에 또 삽질하기 때문에 이력이 있는게 무조건 좋다고 생각해요.
브랜치도 새로 따서 작업했습니다.

저는 Icarus 라는 테마를 선택했습니다. 선택의 이유는 다음과 같습니다.

  • commit 히스토리가 성실하게 기록되어 있다.
  • star(좋아요) 수가 3800 개로 살펴본 테마중 별 개수가 제일 높았다.
  • 물론 디자인도 괜찮다.

테마를 fork 하여 사용하면 다음과 같은 장점이 있습니다.

  • 테마 작성자의 commit 들을 내 브랜치로 merge 가능하다. 즉, 버그들이나 개선점들을 반영할 수 있다.
  • 테마 설정등을 Git History 로 관리되기 때문에 추후 기능의 추가, 삭제, 수정에 이점이 있다.

결론?

사실 GitHub 페이지로 무료 호스팅하여 블로그를 꾸리고 싶다면 정말 필요한 Repository 는 배포를 위한 Repository 입니다.

나머지는 저장소는 없어도 됩니다.. 깃에 대해 잘 모른다면 무슨 말을 하는건지 잘 모르실거라 생각됩니다.
갚이 있게 풀어내지 못해서 죄송합니다.

2년 만에 포스팅하려니 어렵네요 ㅠㅠ
